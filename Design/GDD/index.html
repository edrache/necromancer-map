Necromancer Island

Systemic disruption of local equilibrium.

Game Overview

Genre

Systemic Simulation / God Game

Target Audience

Enthusiasts of emerging systems and simulation games.

Key Features

  • Local Equilibrium: Independent regions balancing Sustain vs Demand.
  • Systemic Disruption: Player acts as a local catalyst for chaos.
  • Visual Decay: Saturation and emoji-based feedback of world state.
  • Emergent Narrative: The story is told through the simulation's recovery or collapse.

Core Gameplay Mechanics

Necromancer Actions

The player introduces disturbances through local killings and spawning Death Sources.

Simulation Loop

Turn-based: Sustain Production → Diffusion → Consumption → Regression/Expansion.

Regional Death Severity

Accumulated death triggers regional repression events and player "Death" time-skips.

Visual & Audio Style

Aesthetic: Dark, minimalist UI with vibrant emoji representing the living world.

  • High Stability: Bright, saturated emojis.
  • Regression: Desaturated colors, "ruin" emoji variants.
  • Chaos: Glitchy animations or color shifts during repression events.

Technical Architecture

Built with Vanilla JavaScript and CSS Grid.

  • Engine: Discrete turn simulation loop.
  • Connectivity: Flood-fill based region partitioning.
  • Efficiency: Reactive DOM updates only on changed cells.

Roadmap

  • MVP: Core simulation, emoji visuals, basic player interaction.
  • V1.1: Sound synthesis based on world stability.
  • V1.2: More complex region events and "Necromancer Spirit" progression.
